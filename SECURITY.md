# Security Policy

## Supported Versions

Security fixes are best-effort and prioritized for the latest `main` branch.

## Reporting a Vulnerability

Please do not disclose vulnerabilities publicly in issues.

When reporting, include:

- A clear description of the issue
- Steps to reproduce
- Potential impact
- Suggested mitigation (if available)

Until private advisory tooling is fully configured, contact the maintainer
listed in `CREDITS.md` and include "[SECURITY]" in the subject/title.

## Response Expectations

- Initial acknowledgement target: within 72 hours
- Triage and impact assessment: as soon as practical
- Fix timeline depends on severity and complexity
